Design guidelines:

1. All UI components must use the liquid glass effect, documentation is at LiquidGlassDocs.md.

2. If a UI component cannot be implemented with Liquid Glass Effect, implement it manually in a way that does not try to mimic liquid glass, but instead just looks modern and simple.

3. NEVER reinvent glass effect yourself. Use the library and docs to implement it where needed.

4. If any type of package is needed, like maps for example, use the apple native IOS ported version, never use a third party version of something where an apple native public version exists.
